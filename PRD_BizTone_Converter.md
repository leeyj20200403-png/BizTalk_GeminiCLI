# PRD: BizTone Converter (업무 말투 변환 솔루션)

**문서 상태: 초안**
**작성일: 2026년 2월 20일**
**작성자: Gemini**

---

## 1. 개요 (Overview)

### 1.1 문서 목적
본 문서는 'BizTone Converter' 프로젝트의 목표, 기능, 사용자 요구사항을 정의하는 것을 목적으로 합니다. 개발팀과 이해관계자 간의 명확한 소통을 위한 기준을 제시하며, 프로젝트의 범위와 방향을 명확히 합니다.

### 1.2 문제 정의 (Problem Statement)
기업 내 많은 직원이 직급이나 관계(상사, 동료, 고객)에 따라 적절한 커뮤니케이션 톤을 유지하는 데 어려움을 겪고 있습니다. 특히 신입사원이나 비즈니스 커뮤니케이션 경험이 적은 경우, 부적절한 말투로 인해 오해를 사거나 전문성이 떨어져 보일 수 있습니다. 이러한 문제를 수동으로 해결하는 데는 시간과 노력이 소모되며, 커뮤니케이션 품질의 일관성을 유지하기 어렵습니다.

### 1.3 목표 (Goals & Objectives)
- **주요 목표**: 사용자의 일상적인 문장을 전문적인 업무용 문장으로 자동 변환하여, 원활하고 효율적인 비즈니스 커뮤니케이션을 지원합니다.
- **세부 목표**:
  1. **대상별 맞춤 변환**: '상사', '타팀 동료', '고객' 세 가지 대상에 최적화된 말투를 제공합니다.
  2. **직관적인 사용성**: 누구나 쉽게 사용할 수 있는 단순하고 직관적인 웹 인터페이스를 구축합니다.
  3. **빠른 응답 속도**: 사용자가 지연을 느끼지 않도록 평균 3초 이내에 변환 결과를 제공합니다.
  4. **신뢰성 있는 서비스**: 안정적인 시스템 운영을 통해 언제든 필요할 때 사용할 수 있는 서비스를 제공합니다.

### 1.4 목표 사용자 (Target Audience)
- **주요 사용자**:
  - 기업의 신입사원 및 주니어 직원
  - 비즈니스 문서 및 이메일 작성에 어려움을 느끼는 직원
- **확장 사용자**:
  - 다양한 이해관계자와 소통하는 팀장 및 관리자
  - 고객 응대가 잦은 영업 및 CS(고객 서비스) 담당자

## 2. 사용자 시나리오 및 요구사항 (User Scenarios & Requirements)

### 2.1 사용자 스토리
1. **신입사원 (As a new employee)**: 나는 상사에게 업무 보고를 할 때, 내 평소 말투가 너무 가볍게 들리지 않도록 정중하고 격식 있는 문장으로 변환하고 싶다. 이를 통해 전문적인 인상을 주고 명확하게 상황을 전달하고 싶다.
2. **타팀 협업 담당자 (As a cross-functional team member)**: 나는 다른 팀에 자료나 업무 협조를 요청할 때, 너무 딱딱하지 않으면서도 예의를 갖춘 명확한 메시지를 보내고 싶다. 이를 통해 긍정적인 협업 관계를 유지하고 싶다.
3. **고객 지원 담당자 (As a customer support specialist)**: 나는 고객의 문의에 답변할 때, 회사를 대표하는 공식적이고 신뢰감 있는 어조를 사용하고 싶다. 이를 통해 고객 만족도를 높이고 회사의 이미지를 제고하고 싶다.

### 2.2 기능적 요구사항

#### FR-1: 핵심 말투 변환 기능
- **FR-1.1**: 사용자는 원본 텍스트를 입력할 수 있어야 한다.
- **FR-1.2**: 텍스트 입력 영역은 최대 500자까지 입력을 허용해야 하며, 현재 글자 수가 표시되어야 한다.
- **FR-1.3**: 사용자는 변환 대상(상사, 타팀 동료, 고객)을 드롭다운 메뉴에서 선택할 수 있어야 한다.
  - **FR-1.3.1**: 드롭다운의 기본 선택값은 '상사'여야 한다.
- **FR-1.4**: '변환하기' 버튼을 클릭하면, 시스템은 입력된 텍스트와 선택된 대상을 기반으로 문장을 변환해야 한다.
- **FR-1.5**: 변환된 결과는 원본 텍스트와 나란히 비교할 수 있도록 별도의 영역에 표시되어야 한다.

#### FR-2: 사용자 인터페이스 (UI)
- **FR-2.1**: 모든 기능(입력, 설정, 결과 확인)은 단일 페이지 내에서 이루어져야 한다.
- **FR-2.2**: 변환된 텍스트를 한 번의 클릭으로 클립보드에 복사할 수 있는 '복사하기' 버튼을 제공해야 한다.
  - **FR-2.2.1**: 복사 성공 시, 사용자에게 시각적인 피드백(예: "복사되었습니다!")을 제공해야 한다.
- **FR-2.3**: 각 대상별 권장 입력 예시를 확인할 수 있는 도움말(툴팁) 기능을 제공해야 한다.
- **FR-2.4**: 데스크톱, 태블릿, 모바일 기기에서 최적화된 화면을 보여주는 반응형 디자인을 적용해야 한다.
  - **FR-2.4.1**: 모바일 화면에서는 원본/결과 패널이 세로로 재정렬되어야 한다.

#### FR-3: 시스템 및 데이터 처리
- **FR-3.1**: 백엔드 시스템은 프론트엔드로부터 텍스트와 대상 정보를 받아 AI 모델(Groq)에 적합한 프롬프트를 생성하여 요청해야 한다.
- **FR-3.2**: AI API 호출 실패 또는 예외 발생 시, 사용자에게 친화적인 오류 메시지(예: "변환에 실패했습니다. 잠시 후 다시 시도해주세요.")를 표시해야 한다.
- **FR-3.3**: 사용자의 개인정보를 제외한 요청 및 변환 성공/실패 여부를 로깅하여 서비스 개선에 활용할 수 있어야 한다.

### 2.3 비기능적 요구사항

- **NFR-1 (성능)**: '변환하기' 버튼 클릭 후 평균 3초 이내에 사용자 화면에 결과가 표시되어야 한다.
- **NFR-2 (사용성)**: 별도의 설명서 없이도 기능을 이해하고 사용할 수 있도록 직관적인 UI/UX를 제공해야 한다.
- **NFR-3 (보안)**: 외부 AI API 키와 같은 민감한 정보는 환경 변수로 관리하며, 클라이언트 측에 노출되어서는 안 된다.
- **NFR-4 (접근성)**: 모든 기능은 키보드만으로 조작 가능해야 하며, 스크린 리더 사용자를 위해 ARIA 레이블을 적절히 사용해야 한다.
- **NFR-5 (안정성)**: API 요청/응답 과정에서 예외 처리를 통해 서비스가 중단되는 일이 없도록 해야 한다.

## 3. 범위 (Scope)

### 3.1 포함될 기능 (In-Scope)
- 텍스트 입력 및 대상(3종) 선택 기능
- AI 기반 텍스트 변환 및 결과 표시 기능
- 결과 텍스트 복사 기능
- 반응형 웹 디자인
- 단일 페이지 애플리케이션(SPA) 형태의 프론트엔드
- RESTful API를 제공하는 경량 백엔드

### 3.2 제외될 기능 (Out-of-Scope)
- 사용자 계정 생성 및 로그인/로그아웃 기능
- 변환 기록 저장 및 조회 기능
- 다국어 지원 (초기 버전은 한국어만 지원)
- 맞춤법 및 문법 교정 기능 (톤앤매너 변환에 집중)
- 변환 결과에 대한 사용자 피드백 수집 및 재학습 기능 (단, '도움됨/안됨' 같은 단순 피드백은 고려 가능)
- 문서 파일(doc, pdf 등) 업로드 및 변환 기능

## 4. 성공 지표 (Success Metrics)
- **사용자 만족도**: 변환 결과물에 대한 '도움됨/안됨' 피드백 비율
- **서비스 활성도**: 일일/주간/월간 변환 요청 수
- **성능**: 평균 변환 응답 시간 (목표: 3초 미만)
- **안정성**: API 호출 성공률 (목표: 99.5% 이상)

---

## 5. 가정 및 제약 조건 (Assumptions and Constraints)
- **가정**: Groq AI API는 한국어 비즈니스 문맥을 이해하고 적절한 수준의 문장 변환을 수행할 수 있다.
- **제약 조건**:
  - 프로젝트는 반드시 명시된 기술 스택(HTML/CSS/JS, Python/Flask, Groq, Vercel)을 사용해야 한다.
  - API 키 등 외부 서비스 자원은 제공된 것을 사용한다.
  - 개발 및 배포는 Vercel 플랫폼을 통해 이루어진다.
